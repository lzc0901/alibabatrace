# AlibabaTrace 驱动驱动开发流程指南

结合当前项目中配置的专属专属技能包（Skills），我们为本项目（Python 3.x Flask 后端、Vue.js 前端、MySQL 数据库、ECharts 可视化）量身定制了以下标准化开发流程。每个阶段都配备了最佳的 AI 辅助话术，便于你随时推进开发。

---

## 阶段一：数据库设计与数据接入（底层基建）

在这个阶段，核心是处理 `data/` 目录下的海量 CSV 文件，并设计高效的数据库结构。

*   **关联能力**：`sql-optimization` (SQL 优化专家)
*   **核心工作**：根据 CallGraph、MSResource 等数据集字段，设计优化的 MySQL 表结构和索引策略，同时规划批量插入和大数据分页查询的最佳实践。

💡 **如果同意，你可以对我说：**
> “用 sql-optimization 技能，帮我分析 `data/` 下各目录 CSV 文件的字段结构，并为它们生成优化的 MySQL 建表语句及数据导入策略。”

---

## 阶段二：后端与前端的基础核心开发（业务实现）

此阶段开始编写具体的 API 服务和前端界面。

*   **关联能力**：`web-coder` (全栈 Web 专家), `create-web-form` (表单专家)
*   **核心工作**：基于 Flask 搭建符合 REST 标准的后端 API；基于 Vue.js 和 ECharts 构建可视化组件；为“算网调度算法验证实验室”开发配置表单与交互式面板。

💡 **如果同意，你可以对我说：**
> “用 web-coder 技能，帮我基于 Flask 生成符合项目规范的 API 蓝图骨架，并用 create-web-form 技能为前端生成一个算法参数配置面板 Vue 组件。”

---

## 阶段三：安全审查与代码重构（质量保障）

业务逻辑基本跑通后，必须确保系统安全且代码易于维护。

*   **关联能力**：`sql-code-review` (SQL 审查专家), `refactor` (重构专家)
*   **核心工作**：专项审查 Flask 中所有的数据库交互层，确保 100% 使用参数化查询，杜绝任何 SQL 注入隐患；将臃肿的路由或组件代码进行解耦重构。

💡 **如果同意，你可以对我说：**
> “用 sql-code-review 和 refactor 技能，审查当前 Flask 代码中的数据库交互操作，解决任何潜在的 SQL 注入风险，并重构过于冗长的业务函数。”

---

## 阶段四：UI/UX 及可视化打磨（前端深度优化）

控制大屏和仪表盘的展示效果。

*   **关联能力**：`web-design-reviewer` (Web 设计审查员)
*   **核心工作**：视觉走查 Vue 应用的响应式布局，修复图表在不同分辨率下的错位问题，优化 ECharts 渲染的大数据量卡顿问题（如开启 dataZoom 或数据降采样）。

💡 **如果同意，你可以对我说：**
> “用 web-design-reviewer 技能检查一下我的看板 Vue 组件，优化一下它的响应式布局结构，并为 ECharts 组件增加大数据量下的性能保护和内存释放策略。”

---

## 阶段五：自动化测试（防退化）

为项目的核心（特别是算网调度算法模块）提供测试覆盖。

*   **关联能力**：`polyglot-test-agent` (多端测试生成专家)
*   **核心工作**：为 Flask 后端接口和核心算法生成全面的 `pytest` 单元测试脚本，特别针对“空图”、“资源打满”等边界条件进行覆盖。

💡 **如果同意，你可以对我说：**
> “用 polyglot-test-agent 技能，为核心算网调度算法和相关的 Flask API 编写涵盖边界条件的单元测试。”

---

## 阶段六：编写规范文档（交付与沉淀）

项目成型后，输出高质量的用户与开发者文档。

*   **关联能力**：`documentation-writer` (Diátaxis 文档撰写专家)
*   **核心工作**：按 Diátaxis 文档框架生成系统接入文档、算法设计白皮书和完整的 API 接口文档规范。

💡 **如果同意，你可以对我说：**
> “用 documentation-writer 技能，参考我们的代码仓库，按照 Diátaxis 框架生成一份完整的后端 API 接口文档与项目操作指南。”
